## Usage

### When to attribute

**Add the footer whenever the agent had a hand in the staged changes** — regardless of how the work was carried out, what fraction the agent wrote versus a tool it drove, or whether a human reviewed or tweaked it afterward. Don't make it a judgment call: if the agent contributed to what's being committed, the footer goes in. Omit it only when none of the diff is the agent's doing, e.g. committing edits a human made entirely on their own.

This footer is the **only** indicator of AI involvement that belongs in the commit. Do not add phrases like `Generated by AI` or `Written with Claude` in the subject, body, or anywhere else.

### Exact value format

The footer value is fixed. Do not paraphrase, abbreviate, or improvise it — two AI-authored commits in the same repo must produce byte-identical footer lines (apart from a deliberate model-version difference). Use exactly:

```
Co-Authored-By: <Agent Name> <noreply@anthropic.com>
```

Rules for the value (the token itself follows the general trailer rules in [features-footers](features-footers.md)):

- **Agent Name**: the agent's product + model name in title case, e.g. `Claude Opus 4.8`, `Claude Sonnet 4.6`. Use the real model you are running as. Do not substitute a bare `Claude`, a tool name like `Claude Code`, a username, or a made-up label. If you genuinely cannot determine the model, fall back to exactly `Claude`.
- **Email**: always `<noreply@anthropic.com>` for Claude — wrapped in angle brackets, no display variations. Do not invent per-commit emails, user emails, or `users.noreply.github.com` addresses.

### Example

```
fix: handle null pointer in user serializer

Closes #88
Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>
```

## Key Points

- Required whenever the agent had a hand in the staged changes, regardless of how the work was produced or whether a human reviewed it; omitted only when none of the diff is the agent's doing
- The footer value is a fixed string: `Co-Authored-By: <Agent Name> <noreply@anthropic.com>`. Only the model name varies, and only because the model actually differs — never because of free-form wording choices
- Email is always `noreply@anthropic.com` for Claude; never a user or per-commit address
- Multiple agents or human co-authors stack as separate `Co-Authored-By` lines, each following the same exact format
- Keep the subject and body free of AI markers — attribution lives only in the `Co-Authored-By` footer
- See [features-footers](features-footers.md) for the general trailer-block placement rules
